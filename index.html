软光渲流程

软光渲流程 场景管理系统识别并创建那些可见的物体. 当所有的物体都创建完毕, 我们遍历每个物体上的几何图元并把他们送往渲染器去绘制图像. void DrawElements() { ApplyVertexShader(); (this->*m_aoDrawFunction[m_pkGeometry->Type])(); } Vertex Shaders 三角形每个点都有位置及一些其他的信息. 假设我们使用动态光照(dynamic lighting), 不仅要计算每个点的点发现, 还需要点的颜色值, 实际上这些颜色值可能还会与贴图的颜色值混合, 这样做点里面又需要贴图的UV坐标了. 一般来说传入 Vertex Program 的是模型空间的坐标, 但并非必须. 该程序最低限度地计算裁剪空间里的点坐标, 法线以及其他的属性, 其结果输出到后面的剪裁和光栅化阶段. void VertexColor3VProgram( in float4 kModelPosition : POSITION, in float3 kModelColor : COLOR, out float4 kClipPosition : POSITION, out float3 kVertexColor : COLOR, uniform float4x4 WVPMatrix) { kClipPosition = mul(kModelPosition, WVPMatrix); kVertexColor = kModelColor; } 上述程序有两个输入(模型空间点坐标, 点色值), 两个输出(剪裁空间点坐标, 点色值), 其中WVPMatrix 表示齐次变换(从模型空间到剪裁空间).

辐射度量学
